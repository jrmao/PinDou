# 图片转换算法增强 - 设计文档

**日期：** 2026-05-02

**状态：** 已批准

---

## 1. 问题描述

v1.1.5 的图片转换算法优化效果不佳，存在以下问题：
1. **线条缺失** - 人物颜色线条变少/消失
2. **主体变白** - 人物主体色彩与背景色相似的部分转化为白色
3. **背景误判** - 浅色主体（如白色衣服）被错误识别为背景

**根本原因：**
- 中值滤波过于激进，将线条误判为噪点
- 背景检测仅基于颜色特征，缺少位置上下文
- 缺少前景主体和背景的区分机制

---

## 2. 改进方案

### 2.1 A部分 - 保守修复（核心）

#### A1. 中值滤波优化
- **问题**：当前 `diffCount >= 6` 太激进，线条像素被误判
- **修复**：提高阈值到 `diffCount >= 7` 才替换，减少误判

#### A2. 背景检测增强
- **问题**：只根据亮度/饱和度判断，没有位置上下文
- **修复**：
  - 边缘区域（靠近画布边缘）：使用背景检测，是背景则填白
  - 中心区域：保守处理，颜色相似也不轻易替换为白色

#### A3. 色彩完整性检查
- **问题**：接近白色的像素被误判
- **修复**：如果一个像素虽然接近白色，但周围存在深色邻居，保留原色

### 2.2 B部分 - 主体识别

#### B1. 简化为两步处理
1. **识别背景区域**：仅在图像边缘的连续区域
2. **主体保持**：主体区域颜色直接匹配，不做背景替换

#### B2. 区域生长标记
- 使用连通区域分析，区分不同色块
- 边缘色块如果是小区域且在边缘，判定为背景

### 2.3 C部分 - 可调参数

| 参数 | 范围 | 默认值 | 说明 |
|------|------|--------|------|
| 去噪强度 | 0-10 | 3 | 控制小区域合并阈值，值越大越保守 |
| 背景敏感度 | 0-10 | 5 | 控制背景判断宽松度，值越大越严格 |
| 边缘锐度 | 0-10 | 5 | 控制线条清晰度，值越大边缘越锐利 |

## 3. 实现计划

### 阶段一：A部分（保守修复）
1. 修复中值滤波阈值
2. 增加背景检测的边缘判断
3. 添加色彩完整性检查

### 阶段二：B部分（主体识别）
1. 实现边缘区域识别
2. 对中心区域和边缘区域分别处理

### 阶段三：C部分（可调参数）
1. 在 UI 添加三个滑块控件
2. 将参数传递给算法函数
3. 实时预览效果
